### Physics verified CORRECT (no action)

- **Werner sampling** (`qstate_core.py:71-90`): ρ = w·|Φ+⟩⟨Φ+| + (1−w)·I/4 decomposes
  exactly into {P(Φ+)=w+(1−w)/4, P(other Bell)=(1−w)/4}; reproduces the state for
  *every* observable including CHSH.
- **QBER = (1−F)/2** in both Z and X bases, and at the E91 key angles π/4, π/2.
- **Measurement math** (`qstate_core.py:41-48,94-133`): U=Ry(−θ) maps the +θ
  eigenvector to |0⟩; collapse/renorm and `_drop_qubit` are an exact partial trace;
  bit-index convention consistent throughout.
- **CHSH quartet** (`e91.py:39-41`): E(a,b)=w·cos(θa−θb) with the stated angles/signs
  gives S = 2√2·F exactly; S≈2 at F=1/√2 confirms the boundary.
- **Sift/key/CHSH disjointness** and **cross-node key-position agreement** are provably
  identical on both sides.
- **Cascade cascade-back** re-checks the current pass, pop-time even-parity skip is
  correct, Alice-parity caching counts each leaked bit once.

### HIGH — FIXED 2026-07-06

- **H1 — `secure_key_bits` double-counted error correction** (`distributed_e91.py`).
  `final_key_bits = key_bits·(1−2H(Q))` already charges an asymptotic H(Q) for EC;
  subtracting the real Cascade `bits_leaked` again gave ≈`1−3H(Q)` (secure rate hit
  zero near QBER 7% instead of 11%). **Fixed**: new `_secure_key_bits()` computes
  `key_bits·(1−H(Q)) − bits_leaked` (PA term + real EC leakage), and 0 until keys
  actually match. `final_key_bits` retained as the labeled asymptotic reference.
- **H2 — in-process `run_session` leaked disclosed sample bits into the key**
  (`e91.py`). `estimate_qber` sampled *random* indices but the key sliced off the
  *first* n_sample, leaving revealed bits in the key. **Fixed**: sample explicitly,
  exclude exactly those positions (mirrors the distributed `key_only` logic). New
  regression test `test_disclosed_sample_bits_excluded_from_key`.

### MEDIUM — open

- **M1** — PLAN announces Alice's bases *before* Bob picks his (`distributed_e91.py`
  ~90 vs ~170), breaking E91's basis-independence: a modified Bob could set
  b_codes = a_codes for 100% sift or fake S. Fix: send `a_codes` in a separate frame
  *after* MEASURE_RESP.
- **M2** — module docstring claims "the raw key never transits the link", but
  MEASURE_RESP carries every raw bit (inherent to the remote-register model). Reword:
  MEASURE_REQ/RESP model the quantum channel and are excluded from the classical-
  eavesdropper set; only RECONCILE/SUMMARY/PARITY are the public discussion.
- **M3** — no key-confirmation after Cascade: an even-weight residual error survives
  silently with `reconciled: true`. Add a SHA-256 digest exchange in RECONCILE_DONE;
  count the extra parity toward bits_leaked.
- **M4** — Cascade BINARY is one block per RPC round trip (~1–2k sequential trips at
  20k pairs; dominates a 30 ms-RTT session). Batch each binary-search *level* into one
  PARITY_REQ across all pending odd blocks.
- **M5** — no cross-node `num_pairs`/`mode` validation: mismatch → IndexError or silent
  desync. Put them in PLAN; Bob adopts `num_pairs = len(plan["surviving"])` and checks
  mode.

### LOW — open

- **L1** — lost B-halves never freed (`_groups` grows ~loss·num_pairs orphans);
  `QuantumStateService.drop` is buggy dead code (leaves id in `g.ids`). Fix drop +
  call it for lost pairs.
- **L2** — RPC has no graceful peer-death handling: 120 s block then uncaught
  `queue.Empty`. Push a sentinel from `Link._rx_loop` on exit; raise `ConnectionError`.
- **L3** — inherited unbounded frame read (`listener.py`, up to 4 GiB from a corrupt
  header). Add a 64 MiB cap (benefits BB84 path too).
- **L4** — `run_sequence_e91` never passes `--seed` → every FABRIC run makes the
  identical "secret" key. Expose a `seed` kwarg; document per-party seeds for
  security-flavored runs.
- **L5** — `chsh_value` returns `abs(S)`, masking a sign bug (would show as S≈−2.8).
  Return signed S; assert S>2.
- **L6** — Cascade with an underestimated QBER (sample missed all errors → qber=0 →
  k1=n): floor `qber = max(qber, 1/(2n))` at the reconcile call site.
- **L7** — the `fidelity` knob is the Werner weight w, not ⟨Φ+|ρ|Φ+⟩ = (3w+1)/4.
  Internally consistent; document in `create_bell_pair`.

### Findings

**HIGH**
- **H1** — wrong repeater theory formula in BOTH comparison scripts
  (`netsquid_comparison.py:145`, `sequence_comparison.py:149`):
  `(1+(2F−1)^(n−1))/2` instead of the correct `(1+3·((4F−1)/3)^(n−1))/4` used
  everywhere else. Baked into comparison JSONs. `paper/PAPER_PLAN.md` §4.2 has the
  wrong one too (main.tex is correct).
- **H2** — SeQUeNCe "repetitions" are identical replicas (hard-coded seed=42) → zero-
  variance error bars; NetSquid uses unseeded global RNG; FABRIC uses SEED+rep. Three
  inconsistent randomness policies in a cross-platform-comparison paper.
- **H3** — sweep runner counts errored runs as "completed" (writes error JSON to the
  same output file, existence = success). Aggregates silently ingest garbage unless
  they filter `result.get('error')`.
- **H4** — `run_all_experiments.py` re-implements sweeps and has drifted: passes no
  seed (vs SEED+rep in standalone scripts) and full ALL_IPS (so chain_length is
  mislabeled as 6 for 2-node runs).

**MEDIUM**: M1 impairment/tuning sweeps use fixed (or absent) seeds → compromised std
(contradicts REPEATER_REVIEW's claim); M2 decoy physics contradicts the security
formula (documented); M3 `mean_rtt` isn't an RTT; M4 `repeater_physical.py` self-check
fails under the current `dephasing` default; M5 fixed 120 s QKD timeout doesn't scale
to 10k qubits; M6 decoy receiver QBER can misalign silently; M7 reconciliation
default-OFF and absent from decoy/relay.
